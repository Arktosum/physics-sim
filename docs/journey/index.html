The Journey

Four reinforcement-learning approaches, tried in order

Each one picked up where the last hit a real ceiling. These are the write-ups from actually building them, in the order it happened — including the wrong turns — rather than cleaned up after the fact into "and then we did the right thing."

  • 1. Q-Learning

    Tabular Q-learning on the double pendulum. Gets surprisingly far with UCB exploration, symmetry augmentation, and reward shaping, but hits a structural wall: the bins^dimensions blowup of discretizing a continuous state space.

  • 2. DQN

    Replaces the table with a neural network — no more discretization of the state — and picks up the two stabilizers that make learning from a network actually converge: a replay buffer and a frozen target network.

  • 3. REINFORCE

    Tackles the other half of the problem — continuous actions, not just continuous state — but vanilla policy gradients turn out to be mathematically unstable on a chaotic system like this: a single lucky episode can send the policy into total, permanent overconfidence.

  • 4. PPO

    The fix for REINFORCE's instability (a trust-region / clipped objective), then a much longer second story once PPO was training correctly: chasing down why the browser stuttered, and a real learning-rate regression that mini-batching quietly introduced.
